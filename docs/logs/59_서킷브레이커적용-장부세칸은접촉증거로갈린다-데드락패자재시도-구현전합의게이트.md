# 59. 서킷 브레이커 적용 — 장부의 세 칸은 접촉 증거로 갈린다 + 데드락 패자 재시도 + 구현 전 합의 게이트

**날짜**: 2026-07-07 (log_58과 같은 날 — arc '재시도 정책' 2·3칸, **arc 닫힘**)
**학습 범위**: 데드락 패자 재시도(Resilience4j `@Retry`)와 토스 서킷 브레이커(`@CircuitBreaker`) 코드 적용. 재방문: log_54(CB 3상태 재발명 — 오늘 코드로), log_56(데드락 패자 씨앗), log_58(transient 축·DEAD 격리 — CB와의 충돌 발견), log_49~50(인바운드/아웃바운드 인터셉터 — 콜스택 혼동 해소). **+ 방법론 사건**: 구현이 코치 템포로 앞서간 위반 → "구현 전 합의 게이트" 규칙 탄생 → 통짜 인출("죽은 토스의 하루")로 빚 회수.

분류: 외부 결제 연동 + DB·트랜잭션 — 코드 적용 + 실전 판단

## 1. 데드락 패자 재시도 — log_58 축의 첫 파생
- 설계는 인출 한 줄로 끝났었다: "transient니까 바운드 걸고 재시도, 넘으면 409." 구체화 4조각(학습자): ⓐ 재시도는 **트랜잭션 밖, 새 트랜잭션으로** — 어노테이션 직감(→ Spring Retry `@Retryable` / Resilience4j `@Retry` 존재 확인) ⓑ `PessimisticLockingFailureException`(형제 커버) 2~3회 즉시(공식 권장 횟수는 **없음** — 데드락은 패자가 죽는 순간 고리가 해체되니 기다릴 이유 없음) ⓒ 소진 시 409 — 단 흔한 결말은 "또 데드락"이 아니라 **성공 or 깨끗한 409**(승자가 커밋했으면 중복 검사가 걸러줌, 학습자 통찰) ⓓ 적용 = 사용자 대면 쓰기 경로(워커는 자체 정책 보유).
- 3택(직접 루프/Spring Retry/R4j): **R4j 채택** — 다음 칸(CB)과 같은 의존성·같은 yml 문법·메트릭. 대가 = tx와의 어스펙트 순서를 우리가 검증해야 함 → **프로브 테스트로 증명**: 1회 패배 후 성공 시 실패 시도의 INSERT가 롤백돼 행 1개(순서가 뒤집혔다면 2개 — 테스트가 잡는다). 리뷰가 jar 소스로 재확인(재시도 order = tx보다 5 낮음 = 바깥).
- 부산물 함정: 주입되는 건 CGLIB **프록시**라 필드 접근은 프록시의 빈 필드를 읽는다 — 상태는 메서드로. / fail-fast 고정 테스트: yml 키가 어긋나면 r4j가 조용히 기본 설정(모든 예외 재시도)으로 폴백 — 락 무관 예외가 1회로 끝남을 박아 설정 드리프트를 회귀 가드로. / 전제 주석: "이 메서드가 최외곽 트랜잭션 경계"(기존 tx 안에서 부르면 죽은 tx에 조인해 전부 실패).

## 2. CB 설계 (전부 학습자 인출·판정)
- **open/closed 어원 뒤집기 사건**: "open = 호출 나감"으로 오독 → 자신의 비유("누전기처럼")로 자가 정정 — **회로가 닫혀야(closed) 전류(호출)가 흐른다**. 단어의 일상 직관(문)이 아니라 전기 회로 관점. 학습자: "단어만 보고 판단하려 해서 생긴 문제."
- 재구성: 트립 = **실패율 AND 최소 표본**(아침 첫 실패 100% 보호) / CB가 지키는 자원 = **우리 스레드**(죽은 토스 × read timeout 5s × 20/s = 동시 100 묶임 — RL은 quota 안이라 통과) / RL=양(정상 가정) vs CB=건강(비정상 가정).
- 설계 4조각(학습자): 건강의 단위 = 토스 전체(인스턴스 1 — 단, 원격 3메서드만. clientKey는 로컬 값이라 창을 성공으로 희석) / "실패" = 아픈 신호만(read timeout·5xx — **4xx는 사용자 오작동으로 열리는 오탐 시나리오까지 도출**) / open 중 사용자 = 503 "일시 중단·돈 안 나감" / 복귀 = 수십초 대기 + 정찰 소수.
- **함정(예고했던 것): CB open ↔ DEAD 격리 충돌.** 토스 10분 장애 시뮬레이션: CallNotPermitted가 워커 catch → recordFailure → 60초×5주기면 멀쩡한 NEEDS_CHECK·REFUND 주문 전부 DEAD + 토스가 살아나도 아무도 수렴 안 시킴. 학습자 결론: **"환경의 실패지 이 주문의 실패가 아니다 — 카운팅하지 않는다."** attempt 카운터가 재는 건 "이 건이 poison인가"고, CB open이 말하는 건 "시스템이 아프다" — 환경 실패를 개별 계좌에 달면 transient(시스템)를 permanent(건)로 오분류. `return`(주기 종료) 채택 — open 중 나머지 건은 소음뿐(대가: 주기 중 half-open 전환 시 정찰 1회를 다음 주기로 미룸 — 60초, 수용).

## 3. 장부의 세 칸 — 접촉 증거로 갈린다 (리뷰 M2 → 토론으로 회수)
- 콜스택 혼동 해소: 아웃바운드 버킷 인터셉터는 인바운드(HandlerInterceptor — 문 앞에서 돌려보냄, 메서드 미실행)가 아니라 **RestClient 안**이다 — 게이트웨이 메서드 **본문 한복판에서** 터지므로 CB(메서드를 감싼 어스펙트)의 시야 안. **인바운드 거부 = 문 앞 / 아웃바운드 거부 = 집 안 — CB는 집 안만 본다.**
- r4j 의미론: **판정자 false = "성공" 계상.** 우리 버킷 거부가 성공 칸에 쌓여 실패율을 희석 — 학습자 재구성: "50건 중 30 버킷차단 + 20 전부 서버에러(실질 100%)인데 CB는 40%로 판단." 하필 장애 때 호출이 토큰을 오래 쥐어 버킷 거부가 **늘어나는** 구조라 왜곡이 최악 시점에 최대.
- 처방 = 세 번째 칸: `ignore-exceptions`(아예 안 적음). **칸의 기준(학습자 봉인)**: 접촉했고 아픈 답 = 실패 / 접촉했고 멀쩡한 답(**4xx 거절도 "정상 작동의 증거"**) = 성공 / **접촉 자체가 없음(우리 사정) = 무시**(증거가 아님).
- 리뷰 M1: `minimum-number-of-calls`는 내부에서 **창 크기로 잘린다**(`min(표본, 창)`) — 테스트 가드(표본 10000)가 조용히 무력화(실효 20). 창도 같이 키워 해결. 교훈: **설정은 선언이 아니라 라이브러리의 해석을 거친다** — "썼다 ≠ 적용됐다"(log_49 자동 동작 함정 부류).
- 다이얼 재사용 공식(학습자 요청으로 정리): 창·최소표본↑=오탐↓·감지 느림 / open 대기↑=죽은 상대를 쉬게·복구 감지 늦음 / 정찰↑=복귀 신중. **"이 값이면 트립까지 최악 얼마를 지불하나"를 계산**(우리: 아픈 호출 10건×5s ≈ 스레드 50초 수업료).

## 한 문장 봉인
> 서킷 브레이커는 **회로의 관점**으로 읽는다(닫혀야 흐른다) — 트립은 실패율 **AND** 최소 표본, 지키는 것은 우리 스레드. 장부의 세 칸(실패/성공/무시)은 **"토스와 접촉했는가, 접촉의 답이 건강 증거인가"**로 갈리고 — 4xx 거절은 성공 칸이 맞고(정상 작동의 증거) 우리 버킷 거부는 무시 칸이다(증거가 아님). 그리고 차단기가 열린 동안의 실패는 **환경의 실패지 개별 건의 실패가 아니다** — 개별 계좌(attempt)에 달면 장애 시간만큼 멀쩡한 주문이 DEAD로 오분류된다.

## 학습법 회고 — 방법론 사건과 회수

### 사건
- 구현 패스가 코치 템포로 앞서갔다 — r4j 배선·설정 의미론(판정자, ignore, cap, 레지스트리 테스트 기법)을 "(d) 문법"으로 뭉뚱그려 **완성 코드로 등장**시킴. 증상: 학습자가 적용 *후에* "yml 값이 뭘 의미하는지 확신이 안 든다", "어떻게 카운팅되는 거야"를 물음 = **코드가 먼저 가고 이해가 뒤따르는 역순**. 사이클도 쉼 없이 이어붙임. 학습자: "너 템포에 끌려가는 느낌. 구현도 혼자 진행했고, 학습에 적합하지 않다."

### 바꿀 것 (다음) — 학습자 확정, 문장 그대로
- **구현 전 합의 게이트**: *"새로 구현하는 개념에 대해서 나에게 먼저 질문해야 해. 토론하면서 의견이 일치하는 지점이 왔을 때 구현을 시작해야 해."* 실행: 구현 전 ① 새로 등장할 개념·결정 목록화 ② 항목별 학습 정도 측정 ③ 처음인 것은 토론 수렴 후에만 타이핑. **리뷰 반영도 같은 게이트**(기계적 수정만 예외). → spiral-learning 스킬 반영(전 프로젝트).

### 회수 (같은 날)
- 학습자 발의: "실수를 회수하기까지는 해야 해. 모른 채로 넘기면 계속 헷갈린다." → **"죽은 토스의 하루" 통짜 인출**(3막: 사망→트립 / 열린 동안 / 복귀)로 코치 주입분을 전부 학습자 서술로 재생 — 빠진 조각 3개(사용자 503·정찰 실패 시 재open·4xx의 칸)만 채우고 마지막 칸 구분은 질문으로 학습자가 완성. **빚 청산 확인.**
